# CLAUDE.md

Project context for Claude Code working in this repo.

## What this project is

**Nik** — an AI life assistant for iOS + Android. The app spans habits, focus/productivity, family circle (multi-user privacy graph), health (HealthKit / Health Connect), money, diary/memory vault, errands, kids view, and an AI chat with voice and GPS awareness. It's design-led: rich theme system (8 universes), gamified pillar score (focus/health/mind/family → 0–1000), voice-first interactions.

The user is non-technical and is using this repo to bring the design from prototype → real shippable mobile app.

## Architecture (current)

**Vite + React 19 + TypeScript** for the app, **Capacitor 8** wrapping the production build into iOS + Android. No Babel-in-browser — fully compiled bundle. Visual fidelity matches the original Claude-generated prototype exactly (same JSX, same CSS, same theme system).

## How to run

**Dev (with HMR, in the browser):**
```bash
cd web && npm run dev
# opens http://localhost:5173
```

**Build + ship to iOS Simulator:**
```bash
cd web && npm run build      # tsc + vite build → /Users/nikhil/NIK/dist/
cd ..  && npx cap sync ios   # copies dist → ios/App/App/public
xcrun simctl launch B2E27E4D-FB6A-435B-B139-2EFA14CADC3A com.nik.app
# or for a clean rebuild + reinstall:
npx cap run ios --target B2E27E4D-FB6A-435B-B139-2EFA14CADC3A
```

**Open the Xcode project to debug native side:**
```bash
npm run open:ios
```

To re-screenshot the running simulator:
```bash
xcrun simctl io booted screenshot /tmp/nik.png
```

To launch the app explicitly on a specific simulator:
```bash
xcrun simctl launch B2E27E4D-FB6A-435B-B139-2EFA14CADC3A com.nik.app
```

To uninstall + redeploy fresh (clears WebView cache):
```bash
xcrun simctl uninstall booted com.nik.app
npx cap run ios --no-sync
```

## How the app is wired

- **Build**: Vite + TypeScript. `tsc -b && vite build` produces a single chunked bundle in `/dist/` (~480KB, ~120KB gzipped). No runtime JSX transform.
- **Themes**: 5 universes defined in `web/src/theme/themes.ts`. `applyTheme(id)` writes 20+ CSS custom properties to `:root` (`--bg`, `--fg`, `--hue`, `--font-display`, etc). `data-theme` and `data-mode` attributes drive cascade overrides in `styles.css`. Switching themes is purely a CSS-variable update — instant, no re-render needed.
- **State**: single `useState<AppState>` in [`web/src/App.tsx`](web/src/App.tsx) holds `{ screen, theme, intensity, density, fontPair, listening, notifVisible, persona, offline }`. `onNav(id)` sets `screen`; renderScreen() switches on `screen` and renders the right component. No router yet — could add `react-router-dom` (already installed) when deep-linking matters.
- **Mock data**: `web/src/data/mock.ts` and `web/src/data/circle.ts`. Will be replaced by API calls when backend lands.
- **Screens** take a common `ScreenProps` (defined in `App.tsx`): `{ dark, intensity, aesthetic, onNav, onVoice, themeId, state, setState, listening?, onExit? }`.

## When making changes

- **Screen tweaks** → edit `web/src/screens/<Name>Screen.tsx`. In dev (`npm run dev`), Vite HMR hot-reloads instantly. For mobile: rebuild + cap sync + relaunch.
- **CSS / theme tweaks** → `web/src/styles.css` for global; `web/src/theme/themes.ts` for theme universe defs.
- **Adding a new screen** → create `web/src/screens/FooScreen.tsx` exporting default `FooScreen`, add `'foo'` to the `ScreenId` union in `App.tsx`, import + register in `renderScreen()`. Optionally add to `TabBar`'s `tabs` or `moreItems` in `web/src/components/shell.tsx`.
- **Adding a primitive / icon** → `web/src/components/primitives.tsx` (export new component); icons go in `web/src/components/icons.tsx` (add to the `I` map).
- **Don't touch `ios/App/App/public/*`** — that's a sync target, overwritten on every `cap sync`.
- **Don't touch `dist/`** — Vite regenerates it.
- **WebView caches aggressively.** If a fresh build isn't showing up, uninstall + reinstall: `xcrun simctl uninstall booted com.nik.app && npx cap run ios --no-sync`.

## Device APIs (later)

We're on Capacitor's plugin ecosystem. When we wire device features, install via:
```bash
npm i @capacitor/geolocation       # GPS / geofencing
npm i @capacitor/push-notifications
npm i @capacitor/camera
npm i @capacitor/preferences       # local storage
# HealthKit / Health Connect: community plugins (less polished than Expo's)
#   - capacitor-health-kit (iOS)
#   - capacitor-health-connect (Android)
```

After `npm i`, run `npx cap sync ios` and Xcode will pick up the native side. Each plugin adds Info.plist entries — Capacitor handles those automatically when configured.

## What's deferred / not yet done

- **Android platform**: only iOS added so far. Add with `npm i @capacitor/android && npx cap add android`.
- **Backend**: no API yet. All data is client-side mock. Architecture decisions (auth, sync, privacy matrix enforcement, AI chat backend) pending — see [docs/Backlog.md](docs/Backlog.md).
- **Real device APIs**: HealthKit, GPS, push, camera, calendar all stubbed in the UI. Capacitor plugins not yet installed (see "Device APIs" section above).
- **Bottom-sheet modals**: prototype's `*-sheets.jsx` (circle-sheets, meds-sheets, diary-sheets) are deferred. CircleScreen has them inlined; others render `null` placeholders. Port when sheet UX matters.
- **State persistence**: nothing survives an app restart (no localStorage / SQLite yet). Add `@capacitor/preferences` when needed.
- **Custom fonts**: Google Fonts load from CDN. For offline / App Store compliance, subset and self-host with `@fontsource/*` packages.
- **Animations**: framer-motion is installed but not yet used. The prototype's CSS keyframes (`fade-up`, `breathe`, `wave-bar`, `xp-fill`) carry most of the motion via styles.css — fine for now.
